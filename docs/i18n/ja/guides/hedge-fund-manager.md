# ヘッジファンドマネージャーガイド — Python を読まずに監督する

資本を運用する。scipy を import する必要はない。**コントロール、ナラティブ、下方境界** が必要。

---

## ファンドに UtahIsMyQuant が与えるもの

| 利益 | 仕組み |
|---------|-----------|
| より速い認識 | ポーリング対 WebSocket sentinel |
| 説明可能な決定 | 各取引に `reason` + `gates_passed/failed` |
| 日中の生存ツール | ストップロス、エクスポージャー上限、サーキットブレーカー |
| 監査証跡 | `AlphaEvent` ログ（チームが配線した場合） |
| バックテスト幻想なし | ライブ挙動への正直さを強制 |

## 与えないもの

- 規制コンプライアンスパッケージ
- 投資家報告テンプレート
- リターン保証
- リスク担当を解雇する理由

## クオントチームへの質問（週次）

1. **ティックの何%が WAIT だった？**
   - 健全: WAIT 高、EXECUTE 低
   - 不健全: 全ティック EXECUTE（ゲートが緩すぎ）

2. **今週のサーキットブレーカー発動回数？**
   - 永遠にゼロ: 閾値が緩すぎるかも
   - 毎日: インフラ問題

3. **Shadow tensor の健全性？**
   - `degradation_score` 上昇傾向 → アルファ劣化 / ノイズ取引

4. **最大のゲート失敗カテゴリは？**
   - シグナル、流動性、リスク上限のどれが問題か

5. **手動でシステムを上書きした？**
   - 週2回超 → システムまたは規律の失敗

## 依頼すべきダッシュボード（最小）

チームは JSONL ログから1日で構築可能:

| パネル | 指標 |
|-------|--------|
| シグナル構成 | `signal` タイプ別カウント |
| アクション構成 | BUY / SELL / EXIT / WAIT |
| ゲート失敗 | `gates_failed` の積み上げ棒 |
| スーパーバイザー | VETO vs FORCE_STOP vs CLEAR |
| 遅延 | p50/p99 `latency_us` |
| PnL | 累積 `pnl_delta`（ペーパー後ライブ） |
| タイス | `tithe_accrued`（ESG ナラティブ用） |

## リスク選好の整合

| 方針 | UtahIsMyQuant ノブ |
|-------------|-------------------|
| アイデアあたり最大2%リスク | `risk_limit=0.02` |
| 銘柄あたり最大10%ブック | `max_position_size=0.10` |
| ポジション5%ストップ | `max_drawdown=0.05` |
| 悪いデータで停止 | `max_latency_ms=200` |

**サインオフ文書:** 有効日付付き1ページにこれらの数値。クオントが週次変更 = 赤旗。

## 「バックテスト」異議への対応

**投資家/取締役の質問:** 「歴史的 Sharpe はどこ？」

**使える回答:**

> 「リアルタイムの幾何安定性と明示的ゲート失敗で運用し、歴史の曲線フィットではありません。ペーパートレードと管理されたライブフェーズが検証です。レガシーバックテストは遅いスリーブ用に残しますが、サブ秒決定は駆動しません。」

**投資家会議では言わない:** 「バックテストは負け犬向け」。リポジトリ README だけで。

## 組織配置

```text
CIO / Manager (you)
    └── Head of Quant (owns Manifold + gates)
    └── Head of Risk (owns enterprise RMS + reviews supervisor logs)
    └── CTO (owns WebSocket infra + broker)
    └── Ops (owns kill switch runbook)
```

UtahIsMyQuant はクオントシグナルと執行の **間** — リスク部門の置き換えではない。

## 予算ナラティブ

| 項目 | ストーリー |
|-----------|-------|
| マーケットデータ | 依然必要；このスタックはフィードを置き換えない |
| 計算 | GPU ML 対 **削減**（CPU ファースト） |
| ベンダーリスク | コンプライアンス用 **除去されない** |
| エンジニアリング | 統合で短期 **増**、カスタム MDH 対長期 **減** |

節約が出たら、すぐレバレッジではなく **データ品質** と **冗長性** に配分。

## その日の運用を止めるとき

次の場合はオペにプラグを引かせる:

1. サーキットブレーカー発動 + 15分で遅延未修正
2. Shadow 劣化が2時間連続で閾値超
3. クオントとリスクの手動上書き争い
4. 相関銘柄で `FORCE_STOP` クラスター（流動性イベント）
