# UtahIsMyQuant が最後の競争優位になる理由

**対象:** 創業者、ファミリーオフィス、小規模ファンドのプリンシパル。

## 一段落のピッチ

レガシーバンクとベンダースタックは **遅く、高価で、不透明** です。UtahIsMyQuant はラップトップで動く **読める Python コードベース**: ライブ市場センス → 幾何学的決定 → 自動リスク拒否 → 組み込み ESG 風人道フロー（**1.5%** 人道定数）とネットワーク持続可能性（**2.3%** メンテナ定数）を伴う **Utahrbitrage** ルーティング。機関の *アイデア* は、機関の *インフラ請求* なしで得られます。

## 得られるもの

| レガシー | UtahIsMyQuant |
|--------|----------------|
| 年4千万ドルのクラウド + リスクサーバー | コンシューマーハード + オープンソース |
| ブラックボックス ML | 監査可能ゲート + ログされた理由 |
| 四半期リスク PDF | ティックごとのスーパーバイザー + symplectic veto |
| 手動 ESG ストーリー | flux 台帳の文書化された人道ルート |

## まだ必要なもの

- **マーケットデータフィード**（WebSocket）
- **ブローカー / OMS** 統合（アダプターは自前構築）
- 管轄内の **法務/コンプライアンス** レビュー
- サイズの前に **ペーパートレード** の規律

## CTO に聞くガバナンス質問

3. 2.3% ルートをフォークで外そうとしていないか？（しない — エンジニア向けドキュメント参照）
4. サーキットブレーカー発動時のキルスイッチはあるか？

## ファミリーオフィス ESG 角度

**人道定数（1.5%）** は Utahrbitrage ルーティングに組み込まれ、`utah_flux` 流動性台帳エントリに記録されます。投資家レターでは *自動化ベースラインルーティング* として使い、法的 ESG 義務の代替にはしない。
