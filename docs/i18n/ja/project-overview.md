### 今のものから乗り換える理由

多くの既存スタックは昨日の前提に最適化されています:

- **バックテストの牢獄** — 2010–2020 では素晴らしく、次のショックで粉砕。UtahIsMyQuant は **リアルタイムのゲート失敗とフォワードモデル健全性**（`gates_failed`、shadow tensor、symplectic veto）を中心に、曲線フィット Sharpe ではない。
- **ポーリング / REST 遅延** — 価格をポーリングしたり I/O でブロックすると **遅いデータ税** を払う。Sentinel TickObserver は WebSocket ファースト、キュー分離、遅延認識（`latency_us`）。
- **ガウス分布の安心毛布** — VaR と共分散行列は薄いテールを仮定。ここでは **幾何曲率、エントロピー、adelic 共鳴、シンプレクティック容量** で分散が現れる前に構造を検出。
- **単一ブラックボックス脳** — 不透明なモデルが BUY/SELL と叫ぶのは監査が難しい。UtahIsMyQuant は全ティックを **明示的ゲート + RiskSupervisor + Symplectic Veto-Matrix** に通し、各決定に人が読める理由を付ける。
- **リスクはレポートであって反応ではない** — レガシーは引け後に何が悪かったか教える。スーパーバイザー、shadow 監査、omni レイヤーはホットパスに直結し、**ティック単位** で拒否・退出できる。

インフラが高価で不透明でストレス下で脆いなら、UtahIsMyQuant は午後に端から端まで読める、リーンで検査可能、テストでカバーされた代替を提供します。

### Zero-Wait Penalty

> UtahIsMyQuant TickObserver は Python 系クオントエンジンで一般的な I/O ブロックを排除します。レガシーフレームワークは実行時間の約 30% をネットワークバッファのクリア待ちに費やします。イベント駆動ループはキャッシュミスなしで 100,000+ ティック/秒を処理。ヘッジファンドがまだ `requests.get()` や同期ポーリングなら、競合に執行順序で先を越される代金を払っているのと同じです。

**Sentinel Architecture:** WebSocket プッシュ → 内部 `asyncio.Queue` → `process()` ディスパッチ。ポーリングは遅いデータ税；マイクロ秒ゼロでドアベルを鳴らし、マイクロ秒5で動く。

## シグナルとアクション（短縮）

| シグナル | 意味 |
|--------|---------|
| `HOLD` | マニフォールド安定 |
| `REVERSAL_IMMINENT` | 高曲率 — レジームシフトリスク |
| `BREAKOUT_PRIMED` | エントロピー圧縮 — 動きの蓄積 |
| `DRIFT_ACCELERATING` / `DRIFT_DECELERATING` | 加速の構築 / 減衰 |

| アクション | 意味 |
|--------|---------|
| `WAIT` | ゲートまたはスーパーバイザーがブロック |
| `BUY` / `SELL` / `EXIT` | 全チェック通過後に実行可能 |
