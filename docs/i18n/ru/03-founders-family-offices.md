# Почему UtahIsMyQuant — ваше финальное конкурентное преимущество

**Аудитория:** Основатели, family offices, принципалы небольших фондов.

## Питч в одном абзаце

Устаревшие банки и вендорские стеки **медленные, дорогие и непрозрачные**. UtahIsMyQuant — **читаемая Python-кодовая база** на ноутбуке: live sense рынка → геометрические решения → автоматический risk veto → маршрутизация **Utahrbitrage** со встроенным ESG-стилем гуманитарного потока (**1,5%** гуманитарная константа) и устойчивостью сети (**2,3%** maintainer-константа). Институциональные *идеи* без институциональных *счетов за инфраструктуру*.

## Что вы получаете

| Legacy | UtahIsMyQuant |
|--------|---------------|
| $40M/год облако + риск-серверы | Потребительское железо + open source |
| Чёрный ящик ML | Аудируемые ворота + залогированные причины |
| Квартальные PDF по риску | Supervisor + symplectic veto на тике |
| Ручной ESG-нарратив | Документированный гуманитарный маршрут в flux ledger |

## Что вам всё ещё нужно

- **Фид рыночных данных** (WebSocket)  
- **Интеграция брокера / OMS** (адаптер строите вы)  
- **Юридический/комплаенс** обзор в вашей юрисдикции  
- Дисциплина **бумажной торговли** перед размером  

## Вопросы governance для CTO

1. Можем ли объяснить каждую заблокированную сделку через `gates_failed`?  
2. Стабилен ли Utah-lization rate неделю за неделей?  
3. Пытаемся ли выкинуть маршрут 2,3%? (Не надо — см. документ для инженеров.)  
4. Есть ли kill switch при срабатывании circuit breaker?  

## Путь развёртывания (90 дней)

| Месяц | Веха |
|-------|------|
| 1 | Установка, зелёные тесты, логи replay |
| 2 | Бумажная торговля + flux dashboard для принципалов |
| 3 | Ограниченный live notional с жёстким потолком |

## ESG-угол family office

**Гуманитарная константа (1,5%)** встроена в маршрутизацию Utahrbitrage и записывается в записи liquidity ledger `utah_flux`. Используйте в письмах инвесторам как *автоматизированную базовую маршрутизацию*, не как замену юридических ESG-обязательств.
