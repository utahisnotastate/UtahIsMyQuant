# Миграция: культура бэктестов → стабильность многообразия в реальном времени

## Вы здесь, если…

- Head of research меряет успех in-sample Sharpe  
- Продакшен-модель ≠ бэктест-модель (классика)  
- «Переоптимизация раз в месяц» в календаре  

## Культурный шок (намеренный)

**UtahIsMyQuant не поставляет фреймворк бэктестинга.**

Это не упущенная функция — заявление дизайна:

> Подгонка истории — подгонка собственного эго. Мы торгуем геометрию *сейчас*.

Кванты будут сопротивляться. Менеджеры спросят «один график с 2020». Готовьте ответы:

1. **Shadow tensor** — здоровье модели вперёд, не backward PnL  
2. **Логи сбоев ворот** — ваш новый research dataset  
3. **Бумажная торговля** — ваш «out of sample»  

## Что заменяет бэктест?

| Старый ритуал | Новый ритуал |
|---------------|--------------|
| Walk-forward grid search | Настройка `sensitivity`, `risk_limit` только на live paper |
| Sharpe за 10 лет | Мониторинг `degradation_score`, срабатываний circuit breaker |
| Slippage model в симе | Измерение `latency_us` + реализованного slippage в paper |
| Research notebook → prod | `omega_point.py` — скелет prod |

## Шаги cutover

### Фаза A — теневой режим (без исполнения)

```python
gen = AlphaGenerator(enable_shadow_audit=True)
# Логировать каждый AlphaEvent.decision 2–4 недели
```

Построить дашборд: счётчики `gates_failed` по имени ворот.

### Фаза B — бумажное исполнение

Подключить paper API брокера к `AlphaEvent.action` только при `supervisor_verdict == "CLEAR"`.

### Фаза C — live micro-notional

Потолок на `risk_limit * capital` по манифесту ворот.

## Сопоставление старых метрик с новыми

| Метрика бэктеста | Прокси в реальном времени |
|------------------|---------------------------|
| Max drawdown | `RiskSupervisor.account_drawdown()` |
| Hit rate | Доля положительных событий `pnl_delta` |
| Turnover | Счётчик `EXECUTE_*` за сессию |
| Alpha decay | `ShadowTensorAudit.degradation_score` |

## Подводные камни

- **Требование паритета** — Старая стратегия не сопоставится 1:1 с сигналами manifold  
- **Перетюнинг sensitivity** на одной волатильной неделе — заморозить параметры минимум на 20 сессий  
- **Игнорирование WAIT** — Большинство тиков должны быть WAIT; это дисциплина
