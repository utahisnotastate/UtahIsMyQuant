# Руководство для управляющего хедж-фондом — надзор без чтения Python

Вы управляете капиталом. Вам не нужно импортировать scipy. Вам нужны **контроль, нарратив и границы downside**.

---

## Что UtahIsMyQuant даёт фонду

| Выгода | Механизм |
|--------|----------|
| Быстрее осознание | WebSocket sentinel vs опрос |
| Объяснимые решения | У каждой сделки есть `reason` + `gates_passed/failed` |
| Инструменты выживания intraday | Stop-loss, потолок экспозиции, circuit breaker |
| Аудит-трейл | Логи `AlphaEvent` (когда команда их подключит) |
| Без фантазий бэктеста | Честность о live-поведении |

## Чего он НЕ даёт

- Пакет регуляторного комплаенса  
- Шаблоны отчётности инвесторам  
- Гарантированную доходность  
- Повод уволить риск-офицера  

---

## Вопросы команде квантов (еженедельно)

1. **Какой % тиков был WAIT?**  
   - Здорово: много WAIT, мало EXECUTE  
   - Нездорово: EXECUTE на каждом тике (ворота слишком мягкие)

2. **Сколько срабатываний circuit breaker на этой неделе?**  
   - Ноль навсегда: порог, возможно, слишком мягкий  
   - Ежедневно: проблема инфраструктуры

3. **Здоровье shadow tensor?**  
   - `degradation_score` растёт → деградация альфы / торговля шумом

4. **Крупнейшая категория сбоев ворот?**  
   - Показывает, проблема в сигнале, ликвидности или risk limit

5. **Перебивали ли систему вручную?**  
   - Чаще двух раз в неделю → сбой системы или дисциплины

## Дашборды, которые стоит запросить (минимум)

Команда может собрать из JSONL за день:

| Панель | Метрика |
|--------|---------|
| Микс сигналов | Счётчик по типу `signal` |
| Микс действий | BUY / SELL / EXIT / WAIT |
| Сбои ворот | Stacked bar `gates_failed` |
| Supervisor | VETO vs FORCE_STOP vs CLEAR |
| Задержка | p50/p99 `latency_us` |
| PnL | Кумулятивный `pnl_delta` (сначала paper, потом live) |
| Десятина | `tithe_accrued` (если ведёте ESG-нарратив) |

## Согласование аппетита к риску

| Ваша политика | Ручка UtahIsMyQuant |
|---------------|---------------------|
| Макс. 2% риска на идею | `risk_limit=0.02` |
| Макс. 10% книги на имя | `max_position_size=0.10` |
| 5% стоп на позициях | `max_drawdown=0.05` |
| Остановка при плохих данных | `max_latency_ms=200` |

**Документ подписания:** Одна страница с этими числами и датой вступления. Кванты меняют еженедельно = красный флаг.

## На возражение про «бэктест»

**Вопрос инвестора / совета:** «Где исторический Sharpe?»

**Ответ, который можно использовать:**

> «Мы работаем на геометрической стабильности в реальном времени и явных сбоях ворот, а не на подгонке исторических кривых. Бумажная торговля и контролируемые live-фазы — наша валидация. Устаревшие бэктесты остаются для медленных рукавов, но не управляют решениями за доли секунды.»

**Не говорите:** «Бэктестинг для неудачников» на встрече с инвесторами. Так можно только в README репозитория.

## Организационное размещение

```text
CIO / Manager (вы)
    └── Head of Quant (владеет Manifold + gates)
    └── Head of Risk (владеет enterprise RMS + ревью логов supervisor)
    └── CTO (владеет WebSocket infra + broker)
    └── Ops (владеет runbook kill switch)
```

UtahIsMyQuant стоит **между** квантовым сигналом и исполнением — не заменяет риск-департамент.

## Бюджетный нарратив

| Статья | История |
|--------|---------|
| Рыночные данные | Всё ещё нужны; стек не заменяет фид |
| Вычисления | **Ниже** vs GPU ML (CPU-first) |
| Вендорский риск | **Не снят** для комплаенса |
| Инженерия | **Выше** краткосрочно на интеграцию, ниже долгосрочно vs bespoke MDH |

Если появляется экономия — направьте в **качество данных** и **резервирование**, не сразу в плечо.

## Когда выключить на день

Приказать ops отключить, если:

1. Circuit breaker сработал + задержка не исправлена за 15 мин  
2. Shadow degradation > порога 2 часа подряд  
3. Спор quant vs risk из-за ручного override  
4. Кластер `FORCE_STOP` на коррелированных именах (событие ликвидности)
